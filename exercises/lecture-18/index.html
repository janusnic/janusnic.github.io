У функції checkIfEmpty ми викликаємо метод trim() для рядка, щоб видалити пробіли на початку та в кінці. Потім ми порівнюємо обрізаний рядок із порожнім рядком, щоб визначити, порожній він чи ні. function checkIfEmpty(str) { if (!str.trim()) { console.log('String is empty'); } else { console.log('String is NOT empty'); } } Метод trim() видаляє всі пробіли з початку та кінця рядка та повертає новий рядок, не змінюючи вихідний. Це дозволяє нам точно обробляти рядки з пробілами. Важливо зауважити, що перевірка порожнього рядка за допомогою оператора заперечення або його поєднання з trim() не оброблятиме нульові та невизначені значення. Якщо вам також потрібно обробити null і undefined, ви можете змінити умову наступним чином: javascriptCopy codefunction checkIfEmpty(str) { if (!str || str.trim() === '') { console.log('String is empty'); } else { console.log('String is NOT empty'); } } У наведеному вище коді умова !str перевіряє, чи є змінна str хибною (null, undefined, порожній рядок або 0). Якщо це так, код всередині блоку if буде виконано, вказуючи, що рядок порожній. Використовуючи ці перевірки, ми забезпечуємо правильну обробку порожніх рядків, у тому числі з пробілами, нульовими значеннями та невизначеними змінними, у нашому коді JavaScript. Крім того, ми можемо використовувати додатковий оператор ланцюжка (?.) для обробки нульових і невизначених значень: const str = ' '; if (str?.trim()) { console.log('The string is NOT empty'); } else { console.log('The string is empty'); } Необов’язковий оператор ланцюжка (?.) замикає і повертає undefined, якщо значення ліворуч є нульовим (нульовим або невизначеним). Це дозволяє нам обробляти випадки, коли рядкова змінна містить невизначене значення: const str = undefined; if (str?.trim()) { console.log('The string is NOT empty'); } else { console.log('The string is empty'); } Замість того, щоб стикатися з помилкою під час спроби викликати метод trim() для невизначеного значення, додатковий оператор ланцюжка запобігає помилці, повертаючи undefined. Використання оператора нульового об’єднання та додаткового оператора ланцюжка може спростити ваш код і ефективніше обробляти сценарії нульового, невизначеного та порожнього рядка. Значення НЕ є порожнім рядком, невизначеним або нульовим Раніше ми дізналися, як перевірити, чи рядок порожній, нульовий або невизначений. Тепер давайте дослідимо, як перевірити, чи рядок не є порожнім рядком, невизначеним або нульовим, використовуючи аналогічний підхід. Щоб визначити, чи значення не є порожнім рядком, невизначеним або нульовим, ви можете скористатися логічним оператором І (&&) у JavaScript. Цей підхід гарантує виконання всіх умов для виконання блоку if. Давайте розберемо код: const str = 'bobbyhadz.com'; if (typeof str !== 'undefined' && str !== null && str.trim() !== '') { console.log('The value is NOT undefined, null, or an empty string'); } else { console.log('The value is undefined, null, or an empty string'); } У наведеному вище коді ми виконуємо такі перевірки: По-перше, ми використовуємо typeof, щоб перевірити, чи змінна str не має тип undefined. Ця умова запобігає викиду кодом помилки, якщо str не було оголошено або визначено. Далі ми перевіряємо, чи str не є нульовим. Нульове значення означає навмисну відсутність будь-якого значення об’єкта. У javascript перевіряючи, чи є рядок нульовим чи порожнім, ми гарантуємо, що змінна має дійсне значення. Нарешті, ми перевіряємо, що str.trim() не є порожнім рядком. Порожній рядок не містить символів і представлений двома лапками без нічого між ними. Поєднуючи ці умови з логічним оператором І (&&), ми гарантуємо, що всі вони мають оцінюватися як істинні для виконання блоку if. Якщо будь-яка з умов не виконується, виконується блок else, вказуючи, що значення є невизначеним, нульовим або порожнім рядком. Використання цього підходу забезпечує повну перевірку, щоб визначити, чи значення має дійсний вміст, допомагаючи уникнути неочікуваної поведінки або помилок у вашому коді.
