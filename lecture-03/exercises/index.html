Місце в ієрархії CSS, де оголошено змінну CSS, визначатиме її рівень видимості на нижчих рівнях ієрархії. Змінна CSS, яка використовується на всій сторінці, оголошується в HTML-елементі html. Змінні CSS, оголошені в HTML-елементі, будуть видимі для споживання його дочірніми елементами. Як правило, змінна CSS видима лише для дочірніх елементів батьківського елемента, у якому вона оголошена. Елемент body є батьківським для своїх дочірніх елементів div, Div 1 і Div 2

Змінна CSS mainColor оголошується в елементі body і використовується в div, щоб встановити зелений фон елементів div. body { --mainColor: limegreen; } div { color: var(--mainColor); } Кажуть, що змінні CSS, оголошені в елементі html, знаходяться в глобальній області видимості. Це означає, що до них можна отримати доступ будь-де в CSS. Чому це працює? HTML-елементі html додається до кореня документа. Усі елементи у веб-документі знаходяться в кореневому елементі, тому змінні CSS, оголошені в HTML-елементі html, каскадно опускатимуться до всіх рівнів документа. Крім того, оголошені змінні CSS будуть видимі для всіх елементів документа. Як і в кореневому HTML-елементі html, змінні CSS також можна оголошувати на всіх рівнях ієрархії CSS або окремому селекторі. Змінні CSS, оголошені на цих рівнях або в селекторі, видимі або обмежені локально для селектора та його дочірніх вузлів.

Змінні CSS можна підняти. Це означає, що змінні CSS можна використовувати до того, як вони будуть оголошені. Змінна CSS --bgColor використовувалася до того, як її було оголошено в HTML-елементі html. Таким чином, змінні CSS можуть бути доступні спочатку, а потім оголошені. Це робить змінні CSS дуже потужною функцією. body { background-color: var(--bgColor); } html { --bgColor: rgb(221, 221, 221); } Вбудовані стилі – це оголошення CSS, які впливають на один елемент HTML, що міститься в атрибуті style. Одна зміна стилю може вимагати кількох редагувань на одній веб-сторінці. По-друге, вбудований CSS також змішує презентаційний код (CSS) з HTML і вмістом, що ускладнює читання та розуміння. Розділення коду та вмісту полегшує обслуговування для всіх, хто працює на веб-сайті. Елемент style містить інформацію про стиль для документа або частини документа. Він містить CSS, який застосовується до вмісту документа, що містить елемент style. Елемент style повинен міститися всередині head документа. Загалом, краще розміщувати свої стилі у зовнішніх таблицях стилів і застосовувати їх за допомогою елементів link. Якщо ви включите кілька елементів style і link у свій документ, вони будуть застосовані до документу у тому порядку, в якому вони включені в документ - переконайтеся, що ви включили їх у правильному порядку, щоб уникнути неочікуваних каскадних проблем. Елемент link визначає зв'язки між поточним документом і зовнішнім ресурсом. Цей елемент найчастіше використовується для посилань на таблиці стилів, але також використовується для встановлення піктограм сайтів (як піктограм стилю «favicon», так і піктограм для головного екрана та програм на мобільних пристроях), серед іншого. Атрибут rel означає «відносини» і є однією з ключових особливостей елемента link - значення вказує на те, як пов’язаний елемент із документом, що містить. Атрибут rel визначає зв'язок між зв'язаним ресурсом і поточним документом. Дійсний для link, a, area і form, можливі значення залежать від елемента, де встановлено атрибут. Тип зв’язків визначається значенням атрибута rel, який, якщо присутній, повинен мати значення, яке є невпорядкованим набором унікальних ключових слів, розділених пробілами. На відміну від імені класу, яке не виражає семантику, атрибут rel повинен виражати лексеми, які семантично дійсні як для машин, так і для людей. Поточні реєстри для можливих значень атрибута rel - це реєстр зв’язків посилань IANA, стандарт HTML Living Standard і сторінка існуючих значень rel у вікі мікроформатів, як запропоновано стандартом Living Standard. Якщо використовується атрибут rel, якого немає в одному з трьох джерел вище, деякі засоби перевірки HTML (такі як Служба перевірки розмітки W3C) створять попередження. Атрибут rel стосується елементів link, a, area і form, але деякі значення мають відношення лише до підмножини цих елементів. Як і всі значення атрибутів ключових слів HTML, ці значення нечутливі до регістру. Атрибут rel не має значення за умовчанням. Якщо атрибут пропущений або якщо жодне зі значень в атрибуті не підтримується, тоді документ не має певного зв’язку з цільовим ресурсом, крім наявності гіперпосилання між ними. У цьому випадку в link і form, якщо атрибут rel відсутній, не має ключових слів, або якщо немає одного чи кількох із наведених вище ключових слів, розділених пробілами, тоді елемент не створює жодних посилань. a і area залишаться створеними посиланнями, але без визначеного зв’язку. Значення атрибута rel=stylesheet дійсне для елемента link, він імпортує зовнішній ресурс для використання як таблицю стилів. Наприклад: At-rules - це оператори CSS, які вказують CSS, як поводитись. Вони починаються зі знака at, '@' (COMMERCIAL AT), після якого йде ідентифікатор і включає все до наступної крапки з комою, ';' (КРАПКА З КОМА), або наступного блоку CSS, залежно від того, що відбудеться раніше. Існує кілька регулярних правил at кожне з яких має інший синтаксис: @charset - визначає набір символів, який використовується таблицею стилів. @import - повідомляє механізму CSS включити зовнішню таблицю стилів. @namespace - повідомляє механізму CSS, що весь його вміст повинен вважатися префіксом простору імен XML. /* Загальна структура */ @identifier (RULE); /* Приклад: вказує браузеру використовувати набір символів UTF-8 */ @charset "utf-8"; Правило @import CSS використовується для імпорту правил стилю з інших таблиць стилів. Правило @import має бути визначене у верхній частині таблиці стилів перед будь-яким іншим правилом at (крім @charset і @layer) і оголошеннями стилів, інакше воно буде проігноровано. @import url; @import url layer; @import url layer(layer-name); @import url layer(layer-name) list-of-media-queries; @import url supports(supports-condition); @import url list-of-media-queries; Де url є рядком або функцією url(), що посилається на місцезнаходження ресурсу для імпорту. URL може бути абсолютним або відносним. Правила @import мають передувати всім іншим типам правил, окрім правил @charset і операторів @layer для створення шару. @import url("my-imported-styles.css"); * { margin: 0; padding: 0; } /* more styles */ Щоб агенти користувачів могли уникнути отримання ресурсів для непідтримуваних типів медіа, автори можуть вказати умови імпорту залежно від медіа. Цей умовний імпорт визначає медіа-запити, розділені комами, після URL-адреси. @import url("fineprint.css") print; @import url("bluish.css") print, screen; @import "common.css" screen; За відсутності медіа-запиту імпорт не залежить від використовуваного медіа. Вказівка all для списку медіа-запитів має той самий ефект. Агенти користувача можуть використовувати функцію supports() у правилі @import, щоб отримати ресурси, лише якщо певний набір функцій підтримується (або не підтримується). Це дозволяє авторам використовувати переваги нещодавно представлених функцій CSS, одночасно забезпечуючи витончені резервні варіанти для старіших версій браузера. Наведені нижче правила @import ілюструють, як можна імпортувати макет, який використовує сітку, якщо підтримується display: grid, або імпортує CSS, який використовує display: flex. Ви також можете використовувати круглі дужки, щоб вказати пріоритет. @import url("gridy.css") supports(display: grid) screen and (max-width: 400px); @import url("flexy.css") supports(not (display: grid) and (display: flex)) screen and (max-width: 400px); Функція url() CSS використовується для включення файлу. Параметр функції - це абсолютна URL-адреса, відносна URL-адреса, URL-адреса blob-файлу або URL-адреса даних. Функцію url() можна передати як параметр інших функцій CSS, як-от функцію attr(). URL-адресою ресурсу може бути зображення, шрифт або таблиця стилів. Існує різниця між URI та URL. URI ідентифікує ресурс. URL-адреса – це тип URI, який описує розташування ресурсу. URI може бути URL-адресою або назвою (URN) ресурсу.
